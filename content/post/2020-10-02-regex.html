 문자형 데이터를 다루는데는 수 많은 기법들이 있습니다. R에서 제공하는 기본 함수들을 통해 처리하기도 하고, 텍스트처리에 특화된 패키지함수를 사용하여 원하는 처리를 하곤 하는데요. 오늘은 그 중에 텍스트데이터를 다루는데 효과적으로 사용되는 정규표현식을 사용하는 법에 대해 알아보도록 하겠습니다.

정규표현식

 정규표현식(regular expression)은 특정한 규칙을 가진 문자열의 집합을 표현하는데 사용하는 형식 언어입니다. 프로그래밍에서 조건문과 반복문을 통해 원하는 결과값을 도출해내는 것처럼, 텍스트데이터에서 조건과 패턴에 맞는 문자열을 찾거나 추출할때 정규식을 사용합니다. 정규표현식에는 특별한 의미를 지니고 있는 메타문자 와 문자 그대로의 의미를 지닌 정규문자 로 구성되어 있습니다.

메타문자

 메타문자는 정규표현식에서 특별한 의미를 갖는 문자 기호입니다. 키보드에서 알파벳과 숫자를 제외한 특수 기호중에서 미리 정의한 기능을 가진 기호들을 메타문자라고 합니다. ! + \ & ^ [] ~ 등의 문자들이 메타 문자이며 이들은 각각 부여받은 기능들이 있어 이를 조합하여 특정한 패턴을 정의하는 방식으로 사용됩니다.

 이해를 돕기위해 예시와 함께 살펴보도록 하겠습니다.

0. 정규표현식 사용자 함수 생성

  R Studio의 stringr패키지 Cheat Sheet에 좋은 예시함수가 있어서 이를 차용하였습니다. 정규표현식 조건에 맞는 문자는 음영처리되어 보여집니다.

see <- function(rx) str_view_all("abc ABC 123\t.!?\\(){}\n abcde aaa bacad .a.aa.aaa abbaab ababcbabcdcbabcde", rx)

1. “^”

 ^(윗꺽쇠)는 문자열의 시작지점을 의미합니다.

see('^') # 문자열의 시작지점을 매칭

2. “$”

 $(달러표시)는 문자열의 끝 또는 문서의 끝을 의미합니다.

see('$') # 문자열의 끝지점을 매칭

3. “.”

 .(마침표)는 임의의 한 글자를 의미합니다.

see('.') # 임의의 글자를 매칭

 다른 규칙없이 마침표만을 단독으로 사용하니 임의의 모든 글자가 선택되었습니다.

4. “?”

  ?(물음표)는 앞에 지정된 문자가 없거나 하나가 있음을 의미합니다.

see('?')
## Error in stri_locate_all_regex(string, pattern, omit_no_match = TRUE, : Syntax error in regexp pattern. (U_REGEX_RULE_SYNTAX, context=`?`)

 물음표를 단독으로 사용하니 에러가 발생하였습니다. 이는 물음표 조건에 맞지 않기 때문입니다. ‘ab?c’(a와 c 사이에 b가 하나 있거나 아니면 아예 없거나) 조건으로 설정해보겠습니다.

see('ab?c') # 'a'와 'c' 사이에 'b'가 하나 있거나 혹은 아예 없는 경우 ('abc' or 'ac')

 정규표현식은 대소문자를 구분하기 때문에 소문자 ’abc’와 ’ac’가 선택된 것을 볼 수 있습니다. 정규표현식의 규칙은 매우 엄격하게 지켜지기 때문에 정규표현을 사용할 때는 정확한 규칙을 만들어서 사용할 필요가 있습니다.

5. “|”

 |(또는)은 의미 그대로 또는을 의미합니다. | 기호를 기준으로 좌우에 해당하는 문자들을 찾습니다.

see('a|c') # 'a' 또는 'c'를 찾아냄

6. “+”

 +(더하기)는 앞 문자가 1개이상 \((1 \geq )\)인 경우를 의미합니다. + 기호의 좌측 문자를 기준으로 찾습니다.

see('a+')  # 'a' 글자가 한개 이상 존재하는 경우  
see('ac+') # 'a' 다음에 'c'가 하나 이상 있음

7. "*"

 * (곱하기)는 앞 문자가 0개이상 \((0 \geq )\)일 경우를 의미합니다.

see('a*') # 'a' 글자를 반환 
see('ac*')# 'a' 다음에 'c'가 없거나 하나 이상 있음

  * (곱하기)는 +(더하기)와 유사하지만 결정적인 차이점으로 +(더하기)는 기호에 해당하는 문자가 하나라도 있는 문자를 반환하지만, * (곱하기)는 앞에 해당문자가 존재하지 않더라도 반환하는 특징이 있습니다.

8. “[ ]”

  [ ](대괄호)는 문자를 묶어서 표현할때 사용됩니다. 대괄호에 묶인 문자들은 각각 or로 취급이 됩니다.

see('[abc]')        # 'a' or 'b' or 'c'를 찾아 반환
see('[abcde]c')     # 'c' 앞에 'a' or 'b' or 'c' or 'd' or 'e' 문자를 찾아 반환 ('ac', 'bc' 반환) 
see('[A-Z][A-Z]C')  # 'C'의 두 부분 앞 문자에서 A부터 Z글자까지 있는 문자를 찾아서 반환 ('ABC' 반환)

 또한, 특정한 문자를 제외하는 의미로서 대괄호 안에 ^(윗 꺽쇠)를 사용합니다.

see('[^abc]') # 'a' or 'b' or 'c'를 제외한 문자를 반환

9. “{ }”

 { }(중괄호)는 문자의 개수를 샐때 사용됩니다. 중괄호에는 세가지 용법이 있습니다.

  • {n,m} : 앞 문자가 n개 이상 m개 이하를 의미.
  • {n,} : 앞 문자가 n개 이상을 의미.
  • {n} : 앞 문자가 정확하게 n개를 의미.
see('a{1,2}') # 'a' 글자가 1개 이상 2개 이하를 표시 
see('b{1,}')  # 'b' 글자가 1개 이상인 경우를 표시
see('c{1}')   # 'c' 글자가 1개인 경우를 표시

10. “( )”

 ( )(소괄호)는 문자를 묶어서 표현할때 사용합니다. 소괄호에 묶인 문자들은 각각 and로 취급됩니다.

see('(abc)')    # 'abc' 문자를 찾아 반화 
see('(abcde)c') # 'c' 앞에 'abcde' 문자를 찾아 반환 (조건에 맞는 결과 없음)
see('a(bc)ba')  # 'a' 와 'ba' 사이에 'bc'가 존재하는 경우의 문자를 반환 ('abcba' 반환)
see('a(b|c)ba') # 'a' 와 'ba' 사이에 'b' 또는 'c'가 존재하는 경우의 문자를 반환 ('abba' 반환 )

 대괄호에 묶인 문자는 ‘or’ 취급이지만, 소괄호는 위의 경우처럼 별도의 or 표시 없이는 ‘and’ 취급이 되는것이 차이점입니다.

11. “\”

 앞서 .(마침표)는 임의의 한 글자를 찾을때 사용하는 메타문자라고 말씀을 드렸는데요. 그렇다면 만약 정규표현식을 사용하여 마침표 자체를 찾고싶을땐 어떻게 해야할까요? ’a.a’라는 문자를 찾는 상황을 가정해보겠습니다.

see('a.a') # 'a.a' 문자 반환?

 ‘a.a’라는 문자를 찾은걸 볼 수 있지만, 그것과 더불어서 ’aaa’, ‘aca’, ’a a’문자 역시 매칭이 되는 것을 볼 수 있습니다. 그 이유는 공교롭게도 “.”(마침표)역시 정규표현식에서 특수한 의미를 갖는 메타문자이기 때문입니다. 정규표현식 내에서 메타문자 자체를 특정하기 위해선 그 문자 앞에 메타문자 종류중 하나인 “\”(역슬레시)를 붙여야합니다.

 그렇다면 이번에는 역슬레시를 활용하여 ’a.a’라는 문자를 찾아보도록 하겠습니다. 문자 가운데에 마침표가 메타문자가 아니라는것을 인지시키기 위하여 마침표 앞에 역슬레시를 붙였습니다.

see('a\.a') # 역슬레시 사용 
## Error: "'a\."로 시작하는 문자열 중에서 '\.'는 인식할 수 없는 이스케이프입니다

 역슬레시를 사용했음에도 불구하고 에러가 발생하였습니다. 여기서 한가지 더 짚고 넘어가야 할 부분은 메타문자를 찾기 위해선 역슬레시를 두개를 붙여야 한다는 것입니다. 왜 그러하냐 하면 역슬레시 또한 “역슬레시 뒤에 붙인 문자는 메타문자가 아니다” 라는 의미를 지닌 메타문자이기 때문입니다. 역슬레시를 하나만 붙인다면 그 붙인 역슬레시역시 메타문자이기 때문에 컴퓨터가 인지하지 못합니다. 역슬레시의 기능을 사용하기 위해선 역슬래시 두 개를 붙여줘야 합니다. 그래야 비로소 컴퓨터가 역슬레시 메타문자가 아님을 인지하고 해당 패턴을 찾아냅니다.

 이처럼 역슬레시가 메타문자를 메타문자로부터 탈출시키기에 이스케이프 문자라 부르기도 합니다.

see('a\\.a') # 역슬레시 두개 사용

 이번에는 찾고자 하던 ’a.a’단어를 정확하게 찾아낸 것을 볼 수 있습니다.

12. “\ + 정규문자”

 앞서 역슬레시( \ )가 메타문자를 탈출시키는 이스케이프 문자라 설명드렸는데요. 정규표현식에서는 이러한 이스케이프 문자의 특성을 살린 특별한 기능이 몇 가지 더 있습니다. 그건 바로 일반 이스케이프 문자와 일반 정규문자와의 조합을 통해 새로운 기능을 수행하는 것입니다. 정규문자중 일부 문자들은 이러한 특성을 지니고 있으며, 이를 정리하자면 다음과 같습니다.

\ + 정규문자 의미 예시 결과
\n 줄바꿈 see(“\\n”)
\t 탭 see(“\\t”)
\s 공백 see(“\\s”)
\S 공백아님 see(“\\S”)
\d 숫자 see(“\\d”)
\D 숫자아님 see(“\\D”)
\w 문자 see(“\\w”)
\W 문자아님 see(“\\W”)
\b 단어경계 see(“\\b”)

13. 특수표현

 이 밖에도 정규표현식에서는 “[: :]” 형태로 존재하는 특수한 표현식이 존재합니다.
[: :] 의미 예시 결과
[:digit:] 숫자 see(“[:digit:]”)
[:alpha:] 문자 see(“[:alpha:]”)
[:lower:] 소문자 see(“[:lower:]”)
[:upper:] 대문자 see(“[:upper:]”)
[:alnum:] 문자+숫자 see(“[:alnum:]”)
[:punct:] 기호 see(“[:punct:]”)
[:graph:] 문자+숫자+기호 see(“[:graph:]”)
[:space:] 띄어쓰기 see(“[:space:]”)
[:blank:] 띄어쓰기+탭 see(“[:blank:]”)

14. 위치 탐색자

  다음은 문자를 기준으로 바로 앞뒤 문자를 찾거나 그 반대인 상황을 찾을때 사용하는 표현식입니다.

see('a(?=c)')  # 'a'가 'c' 바로 뒤에 있는 경우를 반환         
see('a(?!c)')  # 'a'가 'c' 바로 뒤따르지 않는 경우를 반환 
see('(?<=b)a') # 'a'가 'b' 바로 앞에 있는 경우를 반환 
see('(?<!b)a') # 'a'가 'b' 바로 앞에 있지 않는 경우를 반환

 정리를 하다보니 분량이 꽤나 길어진 것 같습니다. 정규표현식은 비단 R에서 뿐만 아니라 다른 프로그래밍 언어에서도 적극 사용되는 용법이어서 한번 익혀두면 두고두고 써먹을 수 있는 장점이 있는데요. 여기에 소개하지 못한 다른 정규표현식도 많이 존재합니다. 더 자세히 공부를 하고자 하시는 분들을 위한 링크를 밑에 삽입하였으니 참고하시면 좋을것 같아요. R에서 텍스트데이터를 다루는 패키지 중 하나인 stringr패키지와 정규표현식을 결합하여 사용하면 더욱 효과적으로 사용할 수 있는데요. 이는 다음장에서 다뤄보도록 하겠습니다.
